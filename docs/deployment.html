Multi-Org Promotion

Move code with evidence, not just enthusiasm. Automate promotion across complex Salesforce environments.

rocket_launch Environment Promotion Logic

1

Dev → QA: Triggered by PR merge. Automatically runs all Apex and LWC tests in the QA sandbox before final schema synchronization.

2

QA → UAT: Requires a "Verified" state in the project record. SFSpeckit validates that all stories have associated Verification Evidence documents before deployment.

3

UAT → PROD: The final gate. Requires BPO sign-off (tracked via /speckit.sf.uat) and passing /speckit.sf.regression tests.

Deployment Command

/speckit.sf.deploy --target [alias] --dry-run

Automated Verification Evidence

Deployment in SFSpeckit is "Evidence-Based." The engine generates a formal document containing:

  • task_alt Apex Test Results (100% Pass)
  • task_alt LWC Jest Test Coverage
  • task_alt PMD 7 Security Scan Reports
  • task_alt Environmental Drift Audit
