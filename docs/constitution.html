Project Constitution

The architectural "North Star" for all Spec-kit automated builds.

I

Metadata-First

Declarative artifacts (Objects, Fields, Layouts) must reside in the Mother Story (Story 00) and be deployed before any coding references exist. This ensures the AI is always building against a verified schema.

II

Bulk Awareness

Non-negotiable bulkification. Every service, trigger, and batch job MUST handle 201+ records. Automated tests must include high-volume data scenarios (PNB pattern) to prevent limits drift.

III

Declarative-First

Logic belongs in Flow unless advanced computation or external integration mandates Apex. The Constitution enforces a "Flow First" evaluation before approving Service-Layer code.

IV

Absolute Security

Secure by Design. All Apex classes must specify with sharing and utilize WITH USER_MODE for DML operations to satisfy architectural security rubric.

V

PNB Test Pattern

Positive, Negative, and Bulk. Every developer story is incomplete without evidence of all three coverage types. This is a hard-gate in the implementation scoring engine.

Refer to README Articles VI-IX for the complete set.
