Tactical Lifecycle

Spec-Driven Development (SDD) is the discipline of forcing the Design to exist before the Code.

The SDD Workflow

1. Specification

The TPO establishes the "North Star" (Constitution) and functional features. The Architect analyzes the "Draft Spec" for technical risks and environmental drift before a single line of code is approved.

2. Technical Planning

The Architect breaks requirements into a metadata-first strategy. This phase calculates blast radius and defines the "Mother Story" (Story 00) to unblock parallel development.

3. Implementation (HITL)

Developers execute stories with Human-in-the-Loop (HITL) verification. The engine performs Auto-Heal loops, but the human Architect signs off on the final code structure and security patterns.

4. Verification & QA

Quality is evidence-based. SFSpeckit generates automated Verification Evidence (test results, lints, performance) that serves as the entry criteria for UAT and PR merges.

Human-in-the-Loop (HITL) Gates

AI power remains a co-pilot. SFSpeckit enforces three critical human gates during the lifecycle:

gavel

Architectural Review

Mandatory review of generated stories to ensure dependency alignment and metadata compliance before Jira ingestion.

verified

Functional Sign-off

BPO (Business Process Owner) verification of UAT scripts generated directly from the tactical specifications.
